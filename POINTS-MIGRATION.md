# Миграция на новую систему баллов (только позиции)

## Что изменилось

С 2026-02-02 баллы начисляются **только за количество позиций**. Скорость работы и эффективность больше не влияют на баллы.

### Тарифы баллов за 1 позицию

| Действие | Склад | Баллы |
|----------|-------|-------|
| Сборка | Склад 1 | 1 |
| Сборка | Склад 2, 3 | 2 |
| Проверка сама (выбрал себя) | Склад 1 | 0,78 |
| Проверка сама (выбрал себя) | Склад 2, 3 | 1,34 |
| Проверка с диктовщиком | Склад 1 | проверяльщик 0,39 / диктовщик 0,36 |
| Проверка с диктовщиком | Склад 2, 3 | проверяльщик 0,67 / диктовщик 0,61 |

---

## Источник формул

**Единственный источник:** `src/lib/ranking/pointsRates.ts`. Все расчёты (updateStats, recalc, aggregateRankings) используют его.

⚠️ Не используйте `stats:calculate` и `stats:recalculate-all` — они применяют устаревшую формулу и перезапишут баллы неверно. Эти скрипты откажутся запускаться при наличии `normVersion: positions-only`.

## Инструкция для production

### 1. Резервное копирование (обязательно)

```bash
# Создать бэкап БД перед пересчётом
npm run db:backup
```

### 2. Деплой кода

```bash
git pull origin main
npm install
npm run build
```

### 3. Пересчёт баллов (dry-run)

Сначала проверьте, сколько записей изменится, **без записи в БД**:

```bash
npm run stats:recalc-points
```

В выводе будет: `Изменено записей: N`. Если всё ок — переходите к применению.

### 4. Применение пересчёта

```bash
npm run stats:recalc-points -- --apply
```

Скрипт:
- обновит `TaskStatistics.orderPoints` и `basePoints` по новой формуле;
- пересчитает `DailyStats` из TaskStatistics;
- пересчитает `MonthlyStats` из DailyStats;
- обновит `dailyRank` и `monthlyRank`.

### 5. Перезапуск приложения

```bash
# Если через pm2:
pm2 restart sklad-spec

# Или systemd, docker и т.п. — по вашей конфигурации
```

### 6. Проверка

```bash
# Аудит баллов за последнюю неделю
npm run stats:audit-week
```

В выводе должно быть: `✅ Аудит пройден.` и `Расхождение: 0`.

## Команды

| Команда | Описание |
|---------|----------|
| `npm run stats:recalc-points` | Dry-run пересчёта (без записи) |
| `npm run stats:recalc-points -- --apply` | Применить пересчёт в БД |
| `npm run stats:audit-week` | Аудит баллов за неделю с понедельника |
| `npm run stats:audit-spot-check` | Выборочная проверка (диктовка/проверка/сборка) |
| `npm run stats:audit-user-points -- "Имя"` | Детальный аудит по пользователю |

**Опции audit-spot-check:** `--7` (последние 7 дней), `--month`, `--30`.

## Что делать, если что-то пошло не так

1. **Восстановить БД из бэкапа:**
   ```bash
   npm run db:restore
   ```

2. **Откатить код:**
   ```bash
   git revert HEAD
   npm run build
   pm2 restart sklad-spec
   ```

3. **Если расхождения в аудите:** повторно запустите `npm run stats:recalc-points -- --apply`.

## Производительность топа / бенчмарк

Замер того же пути, что даёт «Загрузка рейтинга» на `/top` (ядро — `aggregateRankings`):

```bash
npm run benchmark:stats
npm run benchmark:stats:quick
```

Опции: `--quick` (только `week`), `--http` (нужен `npm run dev` — полный HTTP-стек). Жёсткие бюджеты для CI: переменные `STATS_BUDGET_MS_TODAY`, `STATS_BUDGET_MS_WEEK`, `STATS_BUDGET_MS_MONTH`.

**Почему долго:** несколько тяжёлых `findMany` по `task_statistics` + расчёт доп. работы по сессиям месяца. Цель — стабильно &lt; 2–5 с; пока не достигнуто — поднимайте `proxy_read_timeout` у nginx для `/api/statistics/`.

**Дальнейшие шаги:** кэш весов эффективности по 15‑мин ведрам внутри одного `aggregateRankings`; `EXPLAIN QUERY PLAN` и индексы под даты `completedAt`/`confirmedAt`; прогрев кэша после деплоя; при необходимости — фоновый пересчёт снимка вместо полного расчёта на каждый запрос.
