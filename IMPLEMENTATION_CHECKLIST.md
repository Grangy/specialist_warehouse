# Чеклист реализации системы заданий по складам

## Задача
Реализовать систему разбиения заказов на задания по складам с ограничением 35 наименований (SKU) на одно задание.

## Логика работы

### Пример:
- Заказ: 100 наименований (100 разных SKU)
  - Склад 1: 10 наименований → 1 задание (10 наим.)
  - Склад 2: 20 наименований → 1 задание (20 наим.)
  - Склад 3: 70 наименований → 2 задания (35 + 35 наим.)
  - **Итого: 4 задания**

### Правила:
1. Максимальное количество наименований (SKU) в одном задании = 35
2. Задания группируются по складам
3. Если на складе наименований > 35, создается несколько заданий по 35 наименований
4. Проверяющий должен подтвердить ВСЕ задания из одного заказа
5. Только после подтверждения всех заданий заказ отправляется в офис

## Структура данных

### ShipmentTask
```prisma
model ShipmentTask {
  id          String   @id @default(cuid())
  shipmentId  String
  warehouse   String   // "Склад 1", "Склад 2", "Склад 3"
  status      ShipmentStatus @default(new)
  createdAt   DateTime @default(now())
  collectorName String?
  lines       ShipmentTaskLine[]
  shipment    Shipment @relation(...)
}
```

### ShipmentTaskLine
```prisma
model ShipmentTaskLine {
  id          String   @id @default(cuid())
  taskId      String
  shipmentLineId String // Связь с исходной позицией заказа
  qty         Int      // Количество в этом задании
  collectedQty Int?
  checked     Boolean  @default(false)
  task        ShipmentTask @relation(...)
  shipmentLine ShipmentLine @relation(...)
}
```
