# Исправление проблемы с логином в PWA (Production)

## Проблема
В production режиме (`npm run start`) логин не работает в PWA, хотя в dev (`npm run dev`) все работает нормально.

## Причина
В production режиме cookies устанавливаются с флагом `secure: true`, что требует HTTPS. Если PWA работает через HTTP (например, в локальной сети), браузер не устанавливает cookie, и логин не работает.

## Решение

### Вариант 1: Использовать переменную окружения (рекомендуется)

Добавьте в файл `.env.local` или `.env`:

```bash
NEXT_PUBLIC_DISABLE_SECURE_COOKIE=true
```

Это отключит требование HTTPS для cookies и позволит работать через HTTP.

### Вариант 2: Использовать HTTPS

Если у вас есть SSL сертификат, используйте HTTPS. В этом случае cookies будут работать автоматически.

### Вариант 3: Использовать обратный прокси с HTTPS

Настройте nginx или другой обратный прокси с SSL, который будет проксировать запросы к Next.js приложению.

## После изменений

1. Пересоберите приложение:
   ```bash
   npm run build
   ```

2. Запустите production сервер:
   ```bash
   npm run start
   ```

3. Проверьте логин в PWA

## Технические детали

- В dev режиме (`npm run dev`) `secure: false` по умолчанию
- В production режиме (`npm run start`) `secure: true` по умолчанию, если используется HTTPS
- Переменная `NEXT_PUBLIC_DISABLE_SECURE_COOKIE=true` переопределяет это поведение
- Переменная `NEXT_PUBLIC_FORCE_SECURE_COOKIE=true` принудительно включает secure (требует HTTPS)

## Безопасность

⚠️ **Важно**: Отключение secure cookies снижает безопасность, так как cookies могут быть перехвачены при передаче по HTTP. Используйте это только в доверенных сетях или для тестирования.

Для production рекомендуется использовать HTTPS.
