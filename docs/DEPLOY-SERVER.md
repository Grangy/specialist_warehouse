# Развёртывание на сервере: миграции и пересчёт

## 1. Миграции базы данных

**Новых миграций в этой версии нет** — `isNewbie` хранится в `UserSettings.settings` (JSON), штрафы — в `system_settings` (ключ `error_penalty_adjustments`). Схема БД не менялась.

Если нужно применить все миграции с нуля:

```bash
# На сервере в каталоге проекта
cd /path/to/sklad_spec

# Проверить статус миграций
npx prisma migrate status

# Применить все неприменённые миграции
npx prisma migrate deploy

# Сгенерировать Prisma Client
npx prisma generate
```

## 2. Пересборка приложения

```bash
# Установить зависимости (если нужно)
npm ci

# Сборка
npm run build
```

## 3. Перезапуск сервиса

```bash
# Пример для systemd
sudo systemctl restart sklad_spec

# Или через pm2
pm2 restart sklad_spec
```

## 4. Пересчёт статистики (если нужно)

Штрафы и баллы за раздельную сборку считаются автоматически при действиях пользователей. Ручной пересчёт нужен только для старых данных.

### Пересчёт баллов (позиции)

```bash
# Сухой прогон (без изменений)
npx tsx scripts/recalculate-points-positions-only.ts

# С применением изменений
npx tsx scripts/recalculate-points-positions-only.ts --apply
```

### Пересчёт дневной статистики

```bash
npx tsx scripts/recalculate-today-stats.ts
```

### Пересчёт рангов за неделю/месяц

```bash
npx tsx scripts/recalculate-today-top-efficiency.ts
```

## 5. Проверка после деплоя

```bash
# Тесты
npm run test

# Линт и сборка
npm run check
```

## 6. Что появилось в этой версии

- **Новенький** — флаг в настройках сборщика, сниженные штрафы (−1/+1 вместо −3/+3)
- **Штрафы за ошибки** — при «Отправить в офис» и регистрации из админки
- **Раздельная сборка** — баллы за собранные позиции при сбросе/таймауте
- **Отображение «Новенький»** — в админке (пользователи) и в топе

## 7. Инициализация system_settings (если нужно)

Ключ `error_penalty_adjustments` создаётся при первом штрафе. Ручная инициализация не требуется.
