# 🗑️ Удаление всех заказов с сервера

Пошаговая инструкция по удалению всех заказов (и завершенных, и нет) с веб-сервера.

---

## ⚠️ ВНИМАНИЕ

**Это действие необратимо!** После удаления все заказы будут потеряны. Убедитесь, что:
- Вы создали бэкап базы данных
- Это именно то, что нужно сделать
- У вас есть права администратора на сервере

---

## 📋 Пошаговая инструкция

### Шаг 1: Подключение к серверу

```bash
ssh root@77.222.47.184
```

### Шаг 2: Переход в директорию проекта

```bash
cd /opt/specialist_warehouse
```

### Шаг 3: Создание бэкапа базы данных

**ОБЯЗАТЕЛЬНО создайте бэкап перед удалением!**

```bash
# Создаем директорию для бэкапов (если еще нет)
mkdir -p backups

# Создаем бэкап базы данных с текущей датой и временем
cp prisma/dev.db backups/dev.db.backup.$(date +%Y%m%d_%H%M%S)

# Проверяем, что бэкап создан
ls -lh backups/
```

**Пример вывода:**
```
-rw-r--r-- 1 root root 2.5M Jan 15 16:30 dev.db.backup.20250115_163045
```

### Шаг 4: Обновление кода (если нужно)

```bash
# Получаем последнюю версию кода
git pull origin main

# Устанавливаем зависимости (если нужно)
npm install
```

### Шаг 5: Запуск скрипта удаления

```bash
# Запускаем скрипт удаления
npx tsx scripts/delete-all-shipments.ts
```

**Скрипт запросит подтверждение:**
1. Сначала спросит: `Вы уверены, что хотите удалить ВСЕ заказы? (yes/no):`
   - Введите: `yes`

2. Затем попросит ввести: `Введите "DELETE ALL" для подтверждения:`
   - Введите: `DELETE ALL`

**Пример выполнения:**
```

   - Все блокировки (locks)
```

## 🔄 Альтернативный способ: Удаление через SQL

Если скрипт не работает, можно удалить напрямую через SQL:

```bash
# Подключаемся к базе данных
sqlite3 prisma/dev.db

# В SQLite выполняем:
DELETE FROM shipment_task_locks;
DELETE FROM shipment_task_lines;
DELETE FROM shipment_tasks;
DELETE FROM shipment_locks;
DELETE FROM shipment_lines;
DELETE FROM shipments;

# Проверяем результат
SELECT COUNT(*) FROM shipments;
SELECT COUNT(*) FROM shipment_tasks;
SELECT COUNT(*) FROM shipment_lines;

# Выходим
.quit
```

**Или одной командой:**
```bash
sqlite3 prisma/dev.db <<EOF
DELETE FROM shipment_task_locks;
DELETE FROM shipment_task_lines;
DELETE FROM shipment_tasks;
DELETE FROM shipment_locks;
DELETE FROM shipment_lines;
DELETE FROM shipments;
SELECT 'Удалено заказов: ' || COUNT(*) FROM shipments;
EOF
```

## 📊 Что будет удалено

При удалении заказов автоматически удалятся (благодаря каскадному удалению):

1. ✅ **Shipment** - все заказы
2. ✅ **ShipmentLine** - все позиции заказов
3. ✅ **ShipmentTask** - все задания
4. ✅ **ShipmentTaskLine** - все позиции заданий
5. ✅ **ShipmentLock** - все блокировки заказов
6. ✅ **ShipmentTaskLock** - все блокировки заданий

**НЕ будут удалены:**
- ❌ **User** - пользователи останутся
- ❌ **Session** - сессии останутся
- ❌ **RegionPriority** - приоритеты регионов останутся

```bash
# или

# Восстанавливаем бэкап
cp backups/dev.db.backup.20250115_163045 prisma/dev.db

pm2 start sklad-spec
# или
systemctl start specialist-warehouse
```

## ⚠️ Важные замечания

1. **Бэкап обязателен!** Всегда создавайте бэкап перед удалением.

2. **Пользователи не удаляются** - только заказы и связанные данные.

3. **Сессии остаются** - пользователям может потребоваться перелогиниться.

4. **После удаления** система будет готова к созданию новых заказов.

5. **Проверьте права доступа** - убедитесь, что у вас есть права на запись в директорию с БД.

## 🆘 Решение проблем

### Ошибка: "Файл базы данных не найден"

```bash
# Проверьте путь к базе данных
cat .env | grep DATABASE_URL

# Проверьте существование файла
find . -name "*.db" -type f
```

### Ошибка: "Permission denied"

```bash
# Проверьте права доступа
ls -la prisma/dev.db

# Если нужно, измените права
chmod 644 prisma/dev.db
```

### Ошибка: "Database is locked"

```bash
# Остановите приложение перед удалением
pm2 stop sklad-spec
# или
systemctl stop specialist-warehouse

# Подождите несколько секунд и попробуйте снова
```
