# 📊 Документация по аналитике

## Обзор

Система аналитики предоставляет детальные отчеты по работе сборщиков, позволяя отслеживать производительность, эффективность и распределение нагрузки.

## 📋 Доступные отчеты

### 1. Отчет по сборщикам (Dashboard)

Основной дэшборд с возможностью выбора диапазона дат, отображающий:

#### 1.1 Количество заказов, позиций и единиц

Для каждого сборщика отображается:
- **Количество заказов** — общее число завершенных заданий
- **Количество позиций** — общее число собранных наименований
- **Количество единиц** — общее число собранных единиц товара

**Пример:**
```
Николай: 15 заказов, 342 позиции, 1,250 единиц
Иван: 12 заказов, 298 позиций, 980 единиц
```

#### 1.2 Среднее время на 100 позиций

Вычисляется как среднее время, затраченное сборщиком на сборку 100 позиций.

**Формула:**
```
Среднее время на 100 позиций = (Общее время / Общее количество позиций) × 100
```

**Формат отображения:** Время в удобочитаемом формате (часы, минуты, секунды)

#### 1.3 Временной диапазон работы

Отображается:
- **Время начала первой сборки** — когда сборщик начал первый заказ в выбранном периоде
- **Время окончания последней сборки** — когда сборщик завершил последний заказ
- **Общий временной диапазон** — разница между началом и концом

### 2. Дополнительные отчеты

#### 2.1 Распределение заказов (Pie Chart)

Круговая диаграмма, показывающая процентное распределение заказов между сборщиками.

#### 2.2 График производительности (Line Chart)

График среднего времени на 100 позиций для каждого сборщика, позволяющий сравнить эффективность работы.

#### 2.3 Сравнительный анализ (Bar Charts)

Столбчатые диаграммы для визуального сравнения:
- Количества заказов
- Количества позиций и единиц

## 🗄️ Структура данных в БД

### Новые поля в таблице `shipment_tasks`:

| Поле | Тип | Описание |
|------|-----|----------|
| `completed_at` | DateTime | Время завершения сборки (когда задание переведено в `pending_confirmation`) |
| `total_items` | Int | Общее количество позиций в задании |
| `total_units` | Int | Общее количество единиц в задании |
| `time_per_100_items` | Float | Время на 100 позиций в секундах |

### Логика вычисления:

1. **total_items** — количество строк в `shipment_task_lines` для данного задания
2. **total_units** — сумма всех `qty` из `shipment_task_lines`
3. **time_per_100_items** — вычисляется при завершении сборки:
   ```typescript
   timeElapsed = completedAt - startedAt (в секундах)
   timePer100Items = (timeElapsed / totalItems) * 100
   ```

## 🔌 API Endpoints

### GET `/api/analytics/collectors`

Получение аналитики по сборщикам за указанный период.

**Параметры запроса:**
- `startDate` (optional) — начальная дата в формате ISO (YYYY-MM-DD)
- `endDate` (optional) — конечная дата в формате ISO (YYYY-MM-DD)

**По умолчанию:** Последние 30 дней

**Пример запроса:**
```
GET /api/analytics/collectors?startDate=2024-01-01&endDate=2024-01-31
```

**Ответ:**
```json
{
  "success": true,
  "data": [
    {
      "collectorId": "user_id",
      "collectorName": "Николай",
      "collectorLogin": "nikolay",
      "totalTasks": 15,
      "totalItems": 342,
      "totalUnits": 1250,
      "avgTimePer100Items": 1800,
      "avgTimePer100ItemsFormatted": "30 мин",
      "firstTaskStart": "2024-01-01T08:00:00Z",
      "lastTaskEnd": "2024-01-31T18:00:00Z",
      "timeRange": "10 ч 30 мин"
    }
  ],
  "dateRange": {
    "start": "2024-01-01T00:00:00Z",
    "end": "2024-01-31T23:59:59Z"
  }
}
```

## 📊 Визуализация

Дэшборд включает следующие графики:

1. **Bar Chart** — Количество заказов по сборщикам
2. **Bar Chart** — Позиции и единицы по сборщикам (два столбца)
3. **Line Chart** — Среднее время на 100 позиций
4. **Pie Chart** — Распределение заказов

Все графики используют библиотеку **Recharts** и адаптированы под темную тему интерфейса.

## 🎯 Метрики производительности

### Общие метрики дэшборда:

- **Всего заказов** — сумма всех завершенных заданий
- **Всего позиций** — сумма всех собранных позиций
- **Всего единиц** — сумма всех собранных единиц
- **Среднее время** — среднее время на 100 позиций по всем сборщикам

## 🔄 Автоматический сбор данных

Данные для аналитики собираются автоматически при завершении сборки:

1. Когда задание переводится в статус `pending_confirmation`
2. Система вычисляет и сохраняет:
   - `completedAt` — текущее время
   - `totalItems` — количество позиций
   - `totalUnits` — количество единиц
   - `timePer100Items` — время на 100 позиций

## 🚀 Использование

1. Перейдите в админ-панель (`/admin`)
2. Выберите вкладку "Аналитика"
3. Выберите диапазон дат (по умолчанию — последние 30 дней)
4. Просматривайте отчеты и графики
